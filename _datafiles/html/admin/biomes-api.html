Biomes API Reference

REST endpoints for listing, creating, updating, and deleting biome definitions. View / Edit

GET /admin/api/v1/biomes Return all biome definitions

Returns an array of all loaded biome definitions, sorted by BiomeId.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/biomes
200 Success
{ "success": true, "data": [ { "biomeid": "default", "name": "Default", "symbol": "•", "description": "A default biome.", "darkarea": false, "litarea": true, "requireditemid": 0, "usesitem": false, "burns": false }, { "biomeid": "lava", "name": "Lava Fields", "symbol": "~", "description": "Rivers of molten rock flow here.", "darkarea": false, "litarea": true, "requireditemid": 0, "usesitem": false, "burns": true } ] }
POST /admin/api/v1/biomes Create a new biome

Creates a new biome definition. The BiomeId is lowercased automatically. Returns 400 if the ID already exists or validation fails.

FieldTypeDescription
BiomeId requiredstringUnique identifier (e.g. forest). Lowercased on save.
Name requiredstringDisplay name shown in the room editor and to players.
Symbol requiredstringSingle character used on the map (e.g. T).
DescriptionstringAtmospheric description shown to players.
LitAreaboolAlways bright.
DarkAreaboolAlways pitch black; Mutually exclusive with LitArea.
BurnsboolCan catch fire.
RequiredItemIdintItem the player must carry to safely traverse this biome. 0 = none.
UsesItemboolWhen true, the required item is consumed on each entry.
curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{ "BiomeId": "swamp", "Name": "Swamp", "Symbol": "s", "Description": "Murky water and twisted trees surround you.", "LitArea": false, "DarkArea": false, "Burns": false, "RequiredItemId": 0, "UsesItem": false }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/biomes
201 Created
{"success":true,"data":{"biomeid":"swamp","name":"Swamp","symbol":"s",...}}
400 Validation error or duplicate ID
{"success":false,"error":"biome \"swamp\" already exists"}
PATCH /admin/api/v1/biomes/{biomeId} Update a biome

{biomeId} is the biome identifier (e.g. forest). Fields in the request body overwrite the existing definition. The BiomeId is always preserved from the URL.

ParameterInDescription
biomeId requiredpathBiome identifier.
curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{ "Name": "Dense Swamp", "Description": "The air is thick with humidity and the smell of decay.", "Burns": false }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/biomes/swamp
200 Success
{"success":true,"data":{"biomeid":"swamp","name":"Dense Swamp",...}}
404 Not Found
{"success":false,"error":"biome not found: swamp"}
400 Validation error
{"success":false,"error":"biome 'swamp' cannot be both dark and lit"}
DELETE /admin/api/v1/biomes/{biomeId} Delete a biome

{biomeId} is the biome identifier. Removes the YAML file from disk and unregisters the biome from memory. The default biome cannot be deleted.

ParameterInDescription
biomeId requiredpathBiome identifier. Cannot be default.
curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/biomes/swamp
200 Success
{"success":true}
400 Cannot delete default
{"success":false,"error":"the default biome cannot be deleted"}
400 Not Found
{"success":false,"error":"biome \"swamp\" not found"}
