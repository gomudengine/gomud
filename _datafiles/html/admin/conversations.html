How conversations work: Each file is tied to a specific mob (by its numeric ID) in a zone. A file holds one or more conversation scripts. When two mobs meet, the engine picks the least-recently-used script whose Supported map contains the initiator’s name pointing to a list that includes the participant’s name. Use * as a wildcard for either side. Each script is a sequence of turns; every turn is one game round. Prefix each command with #1 (initiator) or #2 (participant) followed by the command to run, e.g. #1 say Hello there! or #2 emote nods politely.
