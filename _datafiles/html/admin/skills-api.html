Skills & Professions API Reference

REST endpoints for browsing, creating, editing, and deleting skill and profession definitions. Mutations require the skills.write permission.

Skills
GET /admin/api/v1/skills Return all skills

Returns a map of skillId → Skill object.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/skills
200 Success
{"success":true,"data":{"brawling":{"SkillId":"brawling","Name":"Brawling","Description":"Hand-to-hand combat techniques.","MaxLevel":4}}}
GET /admin/api/v1/skills/{skillId}/yaml View a skill's raw YAML

Returns the raw YAML datafile contents for a skill.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/skills/brawling/yaml
200 Success
{"success":true,"data":{"yaml":"skillid: brawling\nname: Brawling\n...","path":"skills/brawling.yaml"}}
POST /admin/api/v1/skills Create a new skill skills.write

Creates a new skill. SkillId must be lowercase letters/digits/hyphens and unique. Returns the created skill.

curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"SkillId":"fishing","Name":"Fishing","Description":"Catch fish from bodies of water.","MaxLevel":3}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/skills
200 Success
{"success":true,"data":{"SkillId":"fishing","Name":"Fishing","Description":"Catch fish from bodies of water.","MaxLevel":3}}
400 Validation error
{"success":false,"error":"skill \"fishing\" already exists"}
PATCH /admin/api/v1/skills/{skillId} Update a skill skills.write

{skillId} is the skill's string id. Fields in the request body overwrite the existing skill. The SkillId is immutable and always preserved from the path.

curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{"Name":"Fishing","Description":"Updated description.","MaxLevel":4}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/skills/fishing
200 Success
{"success":true,"data":{"SkillId":"fishing","Name":"Fishing","Description":"Updated description.","MaxLevel":4}}
404 Not found
{"success":false,"error":"skill not found: bogus"}
DELETE /admin/api/v1/skills/{skillId} Delete a skill skills.write

{skillId} is the skill's string id. Removes the skill data file from disk. A skill referenced by any profession cannot be deleted and returns 409 Conflict.

curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/skills/fishing
200 Success
{"success":true,"data":null}
409 In use by professions
{"success":false,"error":"skill \"brawling\" is used by professions: warrior, paladin"}
Professions
GET /admin/api/v1/professions Return all professions

Returns a map of professionId → Profession object. Profession ids may contain spaces.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/professions
200 Success
{"success":true,"data":{"warrior":{"ProfessionId":"warrior","Name":"Warrior","Description":"Front-line fighters.","Skills":["brawling","dual-wield"]}}}
GET /admin/api/v1/professions/{professionId}/yaml View a profession's raw YAML

Returns the raw YAML datafile contents for a profession. Percent-encode spaces in the id (e.g. treasure%20hunter).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/professions/warrior/yaml
200 Success
{"success":true,"data":{"yaml":"professionid: warrior\nname: Warrior\n...","path":"professions/warrior.yaml"}}
POST /admin/api/v1/professions Create a new profession skills.write

Creates a new profession. Every entry in Skills must reference an existing skill id, otherwise the request is rejected.

curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"ProfessionId":"angler","Name":"Angler","Description":"Masters of the rod.","Skills":["fishing"]}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/professions
200 Success
{"success":true,"data":{"ProfessionId":"angler","Name":"Angler","Description":"Masters of the rod.","Skills":["fishing"]}}
400 Unknown skill
{"success":false,"error":"profession references unknown skill: bogus"}
PATCH /admin/api/v1/professions/{professionId} Update a profession skills.write

{professionId} is the profession's string id. Fields in the request body overwrite the existing profession. The ProfessionId is immutable. Unknown skill references are rejected.

curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{"Name":"Angler","Description":"Updated.","Skills":["fishing"]}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/professions/angler
200 Success
{"success":true,"data":{"ProfessionId":"angler","Name":"Angler","Description":"Updated.","Skills":["fishing"]}}
404 Not found
{"success":false,"error":"profession not found: bogus"}
DELETE /admin/api/v1/professions/{professionId} Delete a profession skills.write

{professionId} is the profession's string id. Removes the profession data file from disk.

curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/professions/angler
200 Success
{"success":true,"data":null}
404 Not found
{"success":false,"error":"profession not found: angler"}
