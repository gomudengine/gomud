Scripting API Reference

REST endpoints for script function schema and script validation.

GET /admin/api/v1/scripting/functions Return the script functions schema scripting.read

Returns the full schema of available script callback functions grouped by script type (room, mob, item, spell, buff). Each function includes its name, description, parameters, return semantics, and a ready-to-use code stub.

The response is versioned. When new functions are added the version field increments.

Top-level fields:

FieldTypeDescription
versionintSchema version number.
scriptTypesobjectMap of script type name → type definition. Keys: room, mob, item, spell, buff.

Each script type contains:

FieldTypeDescription
labelstringHuman-readable name (e.g. "Room Script").
descriptionstringBrief description of the script type.
functionsarrayList of function definitions.

Each function definition contains:

FieldTypeDescription
namestringFunction name. May contain a placeholder like {command} for dynamic names.
descriptionstringWhat the function does and when it is called.
paramsarrayParameters: each has name, type, description, and optional typeVariants.
returnSemanticsstringDescribes the meaning of the return value.
extendedTimeoutboolTrue if this function is given extra execution time (e.g. onLoad).
dynamicobject|nullIf non-null, the function name contains a user-specified placeholder. Has placeholder, label, description, inputType.
stubstringReady-to-use code template for this function.
