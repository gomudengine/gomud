Buffs API Reference

REST endpoints for browsing, editing, and deleting buff (status effect) definitions.

GET /admin/api/v1/buffs Return all buff specs and flag descriptions

Returns two maps: specs (buffId → BuffSpec) and flags (flagName → description).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs
200 Success
{"success":true,"data":{"flags":{"poison":"Marks the character as poisoned..."},"specs":{"1":{"BuffId":1,"Name":"Fast Healing",...}}}}
GET /admin/api/v1/buffs/{buffId} Get a single buff spec

{buffId} must be a numeric buff ID.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs/1
200 Success
{"success":true,"data":{"BuffId":1,"Name":"Fast Healing","TriggerRate":"1 round","TriggerCount":10,"Flags":["hidden"]}}
404 Not Found
{"success":false,"error":"buff not found: 999"}
PATCH /admin/api/v1/buffs/{buffId} Update buff properties

Merges the supplied fields into the existing spec. The BuffId in the body is ignored - the path parameter is authoritative. The buff is re-validated (including TriggerRate → RoundInterval conversion) before saving.

curl -s -u admin:password \ -X PATCH \ -H "Content-Type: application/json" \ -d '{"TriggerCount":20,"Flags":["poison","hidden"]}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs/1
200 Success
{"success":true,"data":{"BuffId":1,"Name":"Fast Healing","TriggerCount":20,"Flags":["poison","hidden"],...}}
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"buff not found: 999"}
DELETE /admin/api/v1/buffs/{buffId} Delete a buff and its script

Permanently removes the buff YAML and associated JS script from disk and from the in-memory cache.

curl -s -u admin:password \ -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs/1
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"buff not found: 999"}
GET /admin/api/v1/buffs/{buffId}/script Get the buff's JavaScript

Returns the raw script content. script is an empty string if no script file exists.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs/1/script
200 Success
{"success":true,"data":{"script":"function onStart(actor){...}"}}
404 Not Found
{"success":false,"error":"buff not found: 999"}
PUT /admin/api/v1/buffs/{buffId}/script Replace (or delete) the buff's JavaScript

Writes the provided script string to the buff's .js file. Send an empty string to delete the script file.

curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"script":"function onStart(actor){}"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/buffs/1/script
200 Success
{"success":true}
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"buff not found: 999"}
