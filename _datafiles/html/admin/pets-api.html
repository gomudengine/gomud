Pets API Reference

REST endpoints for listing, creating, updating, and deleting pet type definitions. View / Edit

GET /admin/api/v1/pets Return all pet type definitions

Returns a map of petType → Pet object for every loaded pet type.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/pets
200 Success
{ "success": true, "data": { "wolf": { "type": "wolf", "name": "Wolf", "capacity": 0, "damage": { "diceroll": "1d6", "attacks": 1 }, "statmods": { "strength": 2 }, "buffids": [12] }, "mule": { "type": "mule", "name": "Mule", "capacity": 5, "damage": null, "statmods": null, "buffids": null } } }
POST /admin/api/v1/pets Create a new pet type

Creates a new pet type definition. The type field is the unique identifier and is lowercased automatically. Returns 409 if the type already exists.

FieldTypeDescription
type requiredstringUnique pet type identifier (e.g. wolf). Lowercased on save.
namestringDisplay name shown when the player has not named their pet.
namestylestringOptional color pattern applied to the pet name (e.g. :rainbow).
capacityintNumber of items the pet can carry. 0 = cannot carry items.
damageobjectCombat stats. Fields: diceroll (string, e.g. "1d6"), attacks (int).
statmodsobjectMap of stat name → integer modifier granted to the owner (e.g. {"strength": 2}).
buffidsint[]Buff IDs permanently applied to the owner while this pet is active.
curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{ "type": "wolf", "name": "Wolf", "capacity": 0, "damage": { "diceroll": "1d6", "attacks": 1 }, "statmods": { "strength": 2 }, "buffids": [12] }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/pets
201 Created
{"success":true,"data":{"type":"wolf","name":"Wolf","capacity":0,...}}
400 Bad Request
{"success":false,"error":"type is required"}
400 Conflict - type already exists
{"success":false,"error":"pet type \"wolf\" already exists"}
PATCH /admin/api/v1/pets/{petname} Update a pet type

{petname} is the pet type identifier (e.g. wolf). Fields in the request body overwrite the existing definition. The type is always preserved from the URL.

ParameterInDescription
petname requiredpathPet type identifier.
curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{ "name": "Dire Wolf", "capacity": 2, "damage": { "diceroll": "2d6", "attacks": 2 }, "statmods": { "strength": 4, "speed": 2 } }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/pets/wolf
200 Success
{"success":true,"data":{"type":"wolf","name":"Dire Wolf","capacity":2,...}}
404 Not Found
{"success":false,"error":"pet type \"wolf\" not found"}
DELETE /admin/api/v1/pets/{petname} Delete a pet type

{petname} is the pet type identifier. Removes the YAML file from disk and unregisters the type from memory.

ParameterInDescription
petname requiredpathPet type identifier.
curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/pets/wolf
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"pet type \"wolf\" not found"}
