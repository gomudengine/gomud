Conversations API Reference

REST endpoints for listing, reading, writing, and deleting NPC conversation files. View / Edit

GET /admin/api/v1/conversations List all conversation files

Returns a sorted array of every conversation file on disk. Each entry identifies the file by zone and mob_id; it does not include the conversation content. Use the single-file GET endpoint to retrieve content.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/conversations
200 Success
{ "success": true, "data": [ { "zone": "frostfang", "mob_id": 2 }, { "zone": "frostfang", "mob_id": 9 }, { "zone": "dark_forest", "mob_id": 43 } ] }
GET /admin/api/v1/conversations/{zone}/{mobId} Fetch a single conversation file

Returns the full contents of one conversation file, identified by its zone name and the numeric mob ID. The response includes the zone, mob_id, and a conversations array of script objects.

Each script object has a Supported map and a Conversation array. See the data model section below for details.

ParameterInDescription
zone requiredpathZone name (e.g. frostfang). Spaces are treated as underscores.
mobId requiredpathNumeric mob ID (positive integer).
curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/conversations/frostfang/2
200 Success
{ "success": true, "data": { "zone": "frostfang", "mob_id": 2, "conversations": [ { "Supported": { "guard": ["merchant", "citizen"], "*": ["*"] }, "Conversation": [ ["#1 say Halt! State your business."], ["#2 say Just passing through."], ["#1 emote eyes the traveller suspiciously.", "#1 say Move along then."] ] } ] } }
400 Invalid mob ID
{"success":false,"error":"mobId must be a positive integer"}
404 File not found
{"success":false,"error":"conversation file not found: frostfang/2"}
PUT /admin/api/v1/conversations/{zone}/{mobId} Create or replace a conversation file

Writes the full conversation file for the given zone and mob ID. The request body must be a JSON array of script objects (the conversations array, not the outer envelope). If the file does not exist it is created along with the zone directory; if it does exist it is fully replaced. Returns 201 on creation and 200 on update.

The zone name is lowercased and spaces are converted to underscores before the file is written.

ParameterInDescription
zone requiredpathZone name.
mobId requiredpathNumeric mob ID (positive integer).

Request body — array of script objects. Each object must have:

FieldTypeDescription
Supported requiredobjectMap of initiator name → array of participant names. Use "*" as a wildcard for either side. Names are matched case-insensitively at runtime.
Conversation requiredarray of arraysOrdered list of turns. Each turn is an array of command strings. All commands in a turn execute in the same game round. Each command must be prefixed with #1 (initiator) or #2 (participant) followed by the mob command to run.
curl -s -u admin:password -X PUT \ -H "Content-Type: application/json" \ -d '[ { "Supported": { "guard": ["merchant"], "*": ["*"] }, "Conversation": [ ["#1 say Halt! State your business."], ["#2 say Just passing through."], ["#1 emote eyes the traveller suspiciously.", "#1 say Move along then."] ] } ]' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/conversations/frostfang/2
201 Created (new file)
{ "success": true, "data": { "zone": "frostfang", "mob_id": 2, "conversations": [...] } }
200 OK (file updated)
{"success":true,"data":{"zone":"frostfang","mob_id":2,"conversations":[...]}}
400 Validation error
{"success":false,"error":"conversations[0].Conversation must not be empty"}
DELETE /admin/api/v1/conversations/{zone}/{mobId} Delete a conversation file

Removes the YAML file for the given zone and mob ID from disk and invalidates the in-memory existence cache. If the zone directory becomes empty after deletion it is also removed. The operation is idempotent in the sense that a missing file returns 404 rather than silently succeeding.

ParameterInDescription
zone requiredpathZone name.
mobId requiredpathNumeric mob ID (positive integer).
curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/conversations/frostfang/2
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"conversation file not found: frostfang/2"}

Data Model

Conversation files live under _datafiles/conversations/{zone}/{mobId}.yaml. Each file belongs to one mob (the initiator) and contains one or more independent scripts. At runtime the engine picks the least-recently-used script whose Supported map matches the two participants, then steps through its turns one per game round.

ConversationData (script object)
FieldTypeDescription
Supported map[string][]string Maps a lowercase initiator name to a list of lowercase participant names that may use this script. "*" matches any name on either side. Example: {"guard": ["merchant", "citizen"], "*": ["*"]}. The engine checks the initiator key first, then falls back to the "*" key.
Conversation [][]string Ordered list of turns. Each turn is an array of command strings executed in the same game round. Every command string must begin with #1 (initiator mob) or #2 (participant mob) followed by a space and the mob command to run. Example turn with two simultaneous commands: ["#1 emote nods.", "#2 say Understood."]
