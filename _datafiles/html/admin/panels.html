Panel Layout YAML Reference

Top-level fields

FieldValues / Notes
border "full" — every content row has side borders
"open" — only first and last rows have side borders
Default: "full"
charset "single" — ┌─┐ │ └─┘ (default)
"double" — ╔═╗ ║ ╚═╝
"rounded" — ╭─╮ │ ╰─╯
8-rune literal: TopLeft HorizTop TopRight VertLeft VertRight BottomLeft HorizBottom BottomRight
Example: "╔═╗║║╚═╝"
gapSpaces between panels side by side and between slots.
marginSpaces prepended to every output line. Default: 0
default_color Optional ANSI fg color wrapping the entire output.
Numeric code (e.g. "214") or named alias (e.g. "gold").
Omit to disable.

Layout model

  • slots — top-level horizontal columns, rendered side by side
  • rows — within a slot, panel groups stacked vertically
  • panels — within a row, panels placed side by side left-to-right

Per-panel fields

FieldValues / Notes
idUnique identifier. Used in Go: layout.Panel("id").Add(...). Required.
titleRendered verbatim in the top border. May contain ANSI tags. Omit for a plain border line.
min_widthMinimum inner content width. Total panel width = min_width + 4.
columnsLabel+value pairs per line. Default: 1
column_gapSpaces between columns when columns > 1. Default: 2
charsetPer-panel charset override. Falls back to top-level charset when omitted.

Minimal example

border: open
charset: single
gap: 1
margin: 1

slots:
  - rows:
      - panels:
          - id: info
            title: ' Info '
            min_width: 30
  - rows:
      - panels:
          - id: stats
            title: ' Stats '
            min_width: 40
            columns: 2
            column_gap: 2
