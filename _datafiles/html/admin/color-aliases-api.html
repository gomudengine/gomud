Color Aliases API Reference

REST endpoints for managing named ANSI color aliases.

GET /admin/api/v1/color-aliases Return all color aliases

Returns all currently loaded ANSI color aliases as a flat object mapping alias name to numeric color code (0–255).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/color-aliases
200Success
{"success":true,"data":{"username":11,"mobname":51,"gold":220}}
PATCH /admin/api/v1/color-aliases Set or update an alias

Sets a single alias to the given ANSI 256-color code. If the alias already exists it is overwritten. The change is persisted to ansi-aliases.yaml and aliases are reloaded immediately.

Body fields:

  • alias (string, required) — alias name
  • value (integer 0–255, required) — ANSI color code
curl -s -u admin:password \ -X PATCH \ -H 'Content-Type: application/json' \ -d '{"alias":"username","value":195}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/color-aliases
200Success
{"success":true}
400Bad Request
{"success":false,"error":"value must be between 0 and 255"}
DELETE /admin/api/v1/color-aliases/{alias} Delete an alias

Removes the named alias from ansi-aliases.yaml and reloads aliases. The alias name is URL-encoded in the path.

curl -s -u admin:password \ -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/color-aliases/username
200Success
{"success":true}
404Not Found
{"success":false,"error":"alias not found: username"}
