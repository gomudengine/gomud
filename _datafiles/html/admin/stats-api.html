Stats API Reference

REST endpoints for reading server memory and runtime statistics.

GET /admin/api/v1/stats/memory Return memory usage for all registered subsystems

Returns the output of the internal memory reporter registry. The response is a JSON object whose top-level keys are section names (e.g. "Go", "Rooms", "Mobs"). Each section is itself an object whose keys are metric names and whose values are metric entries.

The unit field on each entry indicates how to interpret memory:

unit valuemeaning of memory
"bytes"Approximate byte count of the data structure.
"count"A plain integer count (e.g. GC runs, goroutines, CPU cores). Do not format as bytes.

The count field is the number of items in the collection when applicable (e.g. number of loaded rooms), and is 0 for metrics that do not track a collection size.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/stats/memory
200 Success

Each section maps metric names to entries. "Go" is always present (registered at startup). Additional sections are registered by their respective packages: "Rooms", "Mobs", "Items", "Users", "Scripting", "Templates", "Connections", "Mapper", "Buffs", "Quests", "Spells", "Races", "Pets", "ColorPatterns", "Conversations", "Parties", "BadInputTracker", "Web", and "Events".
