Room Scripts

Event-driven script hooks attached to individual rooms.

Script Path

Room scripts live alongside their room definition YAML file, with .js replacing .yaml:

Room definition _datafiles/world/default/rooms/frostfang/1.yaml
Room script _datafiles/world/default/rooms/frostfang/1.js

Script State

Global variables declared in a room script persist for the lifetime of the room in memory (i.e. while at least one player is present). For short-lived state use RoomObject.SetTempData(); for state that survives unloads use RoomObject.SetPermData().

Event Hooks

onLoad(room RoomObject)

Called the first time a player enters the room after it is loaded into memory. Ideal for one-time initialization — spawning items, setting up state, or running expensive setup logic. onLoad is given a longer execution timeout than other hooks.

ArgumentTypeDescription
roomRoomObjectThe room being loaded.
 
function onLoad(room) {
    // Initialize room state once
    room.SetTempData('initialized', true);
}
function onLoad(room)
    -- Initialize room state once
    room:SetTempData("initialized", true)
end
onEnter(user ActorObject, room RoomObject)

Called when a player enters the room. Return false to suppress the default room description shown on arrival.

Return value: Return false to suppress the room description. Any other return value (or none) shows it normally.
ArgumentTypeDescription
userActorObjectThe player entering.
roomRoomObjectThe room being entered.
 
function onEnter(user, room) {
    user.SendText('A chill wind greets you as you step inside.');
}
function onEnter(user, room)
    user:SendText("A chill wind greets you as you step inside.")
end
onTryExit(exitName string, user ActorObject, room RoomObject)

Called before a player leaves the room, including during flee. Return false to prevent the movement entirely. The player remains in the room and no exit messages are sent.

Return value: Return false to block the movement. Any other return value (or none) allows it.
ArgumentTypeDescription
exitNamestringThe exit direction or name the player is attempting to use (e.g. "north", "gate").
userActorObjectThe player attempting to leave.
roomRoomObjectThe room being exited.
 
function onTryExit(exitName, user, room) {
    if (exitName === 'north') {
        user.SendText('A magical barrier blocks the northern passage.');
        return false; // block movement
    }
    return true;
}
function onTryExit(exitName, user, room)
    if exitName == "north" then
        user:SendText("A magical barrier blocks the northern passage.")
        return false -- block movement
    end
    return true
end
onTryEnter(user ActorObject, room RoomObject)

Called on the destination room before a player enters it. Return false to block the movement. This fires for both normal movement and combat fleeing. It is not called for admin teleports, spawns, or script-driven MoveRoom calls.

Return value: false = block the movement. Any other return value (or none) allows it.
ArgumentTypeDescription
userActorObjectThe player attempting to enter.
roomRoomObjectThe room being entered (this room).
 
function onTryEnter(user, room) {
    // Require a specific quest to enter
    if (!user.HasQuest('dungeon-key:start')) {
        user.SendText('The dungeon gate is sealed. You need a key.');
        return false;
    }
    return true;
}
function onTryEnter(user, room)
    -- Require a specific quest to enter
    if not user:HasQuest("dungeon-key:start") then
        user:SendText("The dungeon gate is sealed. You need a key.")
        return false
    end
    return true
end
onExit(user ActorObject, room RoomObject)

Called when a player leaves the room.

ArgumentTypeDescription
userActorObjectThe player leaving.
roomRoomObjectThe room being exited.
 
function onExit(user, room) {
    room.SendText(user.GetCharacterName(true) + ' departs into the darkness.');
}
function onExit(user, room)
    room:SendText(user:GetCharacterName(true) .. " departs into the darkness.")
end
onIdle(room RoomObject)

Called each game round while at least one player occupies the room. Returning true prevents the engine's default idle actions from running.

Return value: Return true to suppress default idle behavior; false or nothing to allow it.
ArgumentTypeDescription
roomRoomObjectThe room ticking.
 
function onIdle(room) {
    // 10% chance each round to drip water sounds
    if (Math.random() < 0.1) {
        room.SendText('Water drips somewhere in the darkness.');
    }
}
function onIdle(room)
    -- 10% chance each round to drip water sounds
    if math.random() < 0.1 then
        room:SendText("Water drips somewhere in the darkness.")
    end
end
onCommand(cmd string, rest string, user ActorObject, room RoomObject)

Called whenever any player in the room types any command — including invalid ones. Return true to consume the command (mark it as handled and stop further processing). Return false to let the command continue normally.

Return value: true = consumed (stop processing). false or nothing = pass through.
ArgumentTypeDescription
cmdstringThe command word, e.g. "look", "drop", "west".
reststringEverything typed after the command word.
userActorObjectThe player who typed the command.
roomRoomObjectThe current room.
 
function onCommand(cmd, rest, user, room) {
    if (cmd === 'knock' && rest === 'door') {
        room.SendText('A hollow echo rings through the chamber.');
        return true; // consumed
    }
    return false;
}
function onCommand(cmd, rest, user, room)
    if cmd == "knock" and rest == "door" then
        room:SendText("A hollow echo rings through the chamber.")
        return true -- consumed
    end
    return false
end
onCommand_{command}(rest string, user ActorObject, room RoomObject)

A targeted variant of onCommand. Replace {command} with the exact command you want to intercept — for example, define onCommand_look to handle only the look command. When a specific handler is defined, the generic onCommand is not called for that command.

Return value: Same as onCommand — true to consume, false to pass through.
ArgumentTypeDescription
reststringEverything typed after the command word.
userActorObjectThe player who typed the command.
roomRoomObjectThe current room.
 
// Intercept only the 'look' command
function onCommand_look(rest, user, room) {
    user.SendText('You see strange runes carved into the walls...');
    return false; // still show normal room description
}
-- Intercept only the 'look' command
function onCommand_look(rest, user, room)
    user:SendText("You see strange runes carved into the walls...")
    return false -- still show normal room description
end
