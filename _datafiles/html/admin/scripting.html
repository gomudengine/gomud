Scripting Overview

Server-side scripting reference for GoMud world builders.

Scripts can be written in JavaScript or Lua. JavaScript runs as ECMAScript 5.1 via the Goja engine — modern syntax (arrow functions, let/const, template literals) is not available, so use var and standard ES5 patterns. Lua runs as Lua 5.1 (plus goto) via the gopher-lua engine. The language is chosen per script by file extension (.js or .lua), selected in the script editor. Use the JavaScript / Lua toggle on the reference pages to view every example in either language.

Script Objects Overview

Scripts vs. Objects — What is the difference?

Scripts

A script is a JavaScript or Lua file attached to a game entity — a room, mob, item, buff, spell, or pet. It defines event hooks (functions like onEnter or onIdle) that the server calls when something happens. Scripts are the where and when: which entity reacts, and to which event.

Objects

An object is a live handle to a game entity passed into your script as an argument. It exposes methods you call to inspect or manipulate that entity — move it, send it messages, give it items, and more. Objects are the what and how: what you can touch, and how you interact with it.

👤 ActorObject Users & Mobs

The most-used object. Represents any living entity — a player character or an NPC. Lets you read stats, send messages, run commands, give or take items, apply buffs, and much more.

How to get one Passed as a hook argument, or via GetUser(userId) / GetMob(instanceId)
Room Mob Item Buff Spell Pet
🏛️ RoomObject Rooms

Represents a room in the world. Lets you inspect and modify room properties, manage exits and containers, spawn mobs and items, send messages to everyone present, and create temporary rooms.

How to get one Passed as a hook argument, or via GetRoom(roomId)
Room Mob Item Buff Spell Pet
⚔️ ItemObject Items

Represents a single item instance. Lets you read item properties, inspect enchantments, and is passed to item script hooks when the item is found, lost, purchased, or targeted by a command.

How to get one Passed as a hook argument, or via CreateItem(itemId)
Item Room Mob
🐾 PetObject Pets

Represents a player’s companion pet. Lets you read pet type and name, manage the pet’s inventory, and is the primary object in pet script hooks. Returns null if the actor has no pet.

How to get one Via ActorObject.GetPet()
Pet Room Mob
👥 PartyObject Groups

Represents a collection of actors (a player’s party). Most methods mirror ActorObject but apply to every member simultaneously — useful for broadcasting messages or granting rewards to a whole group at once.

How to get one Via ActorObject.GetParty(), GetPartyPresent(), or GetPartyMissing()
Room Mob Spell
📦 ContainerObject Rooms

Represents a named container inside a room (a chest, barrel, crate, etc.). Lets you inspect and modify its contents, check or change its lock state, and manage gold stored inside.

How to get one Via RoomObject.GetContainers()
Room Mob

Time Periods

Whenever a function accepts a time period string, use one of these formats:

FormatExampleNotes
{num} {unit} "3 hours" In-game time. Default when no qualifier is given.
{num} real {unit} "1 real day" Real-world time, not scaled to game speed.
Available units
rounds"10 rounds"Raw game rounds (fastest unit).
minutes"30 minutes"In-game minutes.
hours"2 hours"In-game hours.
days"1 day"One full in-game day cycle.
weeks"1 week"7 in-game days.
months"1 month"~30 in-game days.
years"1 year"365 in-game days.
Anchor-relative units (snap to last occurrence)
noon"1 noon"Last in-game noon + N game days.
midnight"1 midnight"Last in-game midnight + N game days.
sunrise"2 sunrises"Last sunrise + N game days.
sunset"1 sunset"Last sunset + N game days.

Special Targeting Symbols

Use these prefixes in ActorObject.Command() calls for precise targeting when multiple matching names exist in a room:

SymbolTargetsExample
!number Specific ItemId "drop !123" — drops the item with ItemId 123
#number Specific Mob Instance Id "kick #98" — kicks mob instance 98
@number Specific UserId "give !123 to @5" — gives item 123 to user 5

You can also combine them: "give !123 to #98" gives item 123 to mob instance 98. Regular name matching still works when you don't need precision.
