Item Scripts

React to items being picked up, dropped, purchased, or targeted by player commands.

Script Path

Item scripts live alongside the item's YAML definition, with .js replacing .yaml:

Item definition _datafiles/world/default/items/other-0/6-sleeping_bag.yaml
Item script _datafiles/world/default/items/other-0/6-sleeping_bag.js

Event Hooks

onLost(user ActorObject, item ItemObject, room RoomObject)

Called when a player loses the item from their backpack (e.g. dropping it). Changes made to the item inside this hook are not saved.

ArgumentTypeDescription
userActorObjectThe player losing the item.
itemItemObjectThe item being lost.
roomRoomObjectThe room where it happened.
function onLost(user, item, room) {
    user.SendText('The sleeping bag rolls away from you.');
}
onFound(user ActorObject, item ItemObject, room RoomObject)

Called when a player gains the item in their backpack.

ArgumentTypeDescription
userActorObjectThe player gaining the item.
itemItemObjectThe item being gained.
roomRoomObjectThe room where it happened.
function onFound(user, item, room) {
    user.SendText('The sleeping bag feels surprisingly warm.');
}
onCommand(cmd string, user ActorObject, item ItemObject, room RoomObject)

Called whenever a player types any command that targets the item by name. For example, feel bag would trigger this for an item named "bag". Return true to consume the command.

Return value: true = consumed. false or nothing = pass through.
ArgumentTypeDescription
cmdstringThe command word, e.g. "rub", "touch", "activate".
userActorObjectThe player using the command.
itemItemObjectThe item targeted.
roomRoomObjectThe current room.
function onCommand(cmd, user, item, room) {
    if (cmd === 'shake') {
        user.SendText('You shake the bag. Nothing happens.');
        return true;
    }
    return false;
}
onCommand_{command}(user ActorObject, item ItemObject, room RoomObject)

Targeted variant of onCommand. Replace {command} with the exact command to intercept. When defined, the generic onCommand is not called for that command.

Return value: true = consumed. false or nothing = pass through.
ArgumentTypeDescription
userActorObjectThe player using the command.
itemItemObjectThe item targeted.
roomRoomObjectThe current room.
// Fires only when player types 'sleep bag'
function onCommand_sleep(user, item, room) {
    user.GiveBuff(12, 'item'); // rested buff
    user.SendText('You curl up in the sleeping bag and drift off...');
    return true;
}
onPurchase(user ActorObject, item ItemObject, room RoomObject)

Called after a player successfully buys the item from a shop. Return false to prevent the item from being given to the player — useful for scripted purchases that grant something other than the item itself (travel tickets, extra lives, etc.).

Return value: false = don't give the item to the player. Any other return = give normally.
ArgumentTypeDescription
userActorObjectThe buying player.
itemItemObjectThe purchased item.
roomRoomObjectThe shop room.
function onPurchase(user, item, room) {
    // Treat this as a "ferry ticket" — teleport instead of giving item
    user.MoveRoom(500);
    user.SendText('The ferryman nods and you board the boat.');
    return false; // don't give the item
}
onTryPurchase(user ActorObject, item ItemObject, room RoomObject)

Called after the player has been confirmed to have enough gold, but before any gold is deducted or stock removed. Return false to block the purchase entirely — no gold is spent and the item is not removed from the shop.

Return value: false = block the purchase. Any other return value (or none) = allow it.
ArgumentTypeDescription
userActorObjectThe player attempting the purchase.
itemItemObjectThe item being purchased.
roomRoomObjectThe shop room.
function onTryPurchase(user, item, room) {
    // Only allow purchase during daytime
    if (UtilGetTime().Night) {
        user.SendText('The shop is closed at night.');
        return false;
    }
    return true;
}
