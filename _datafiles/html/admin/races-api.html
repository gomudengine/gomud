Races API Reference

REST endpoints for browsing, creating, editing, and deleting race definitions.

GET /admin/api/v1/races Return all races

Returns a map of raceId → Race object.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/races
200 Success
{"success":true,"data":{"1":{"RaceId":1,"Name":"Human","Size":"medium",...}}}
POST /admin/api/v1/races Create a new race

Creates a new race. The server assigns the next available RaceId. Returns the created race.

curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"Name":"Goblin","Description":"A small green creature.","Size":"small","TNLScale":0.9,"UnarmedName":"claw","Damage":{"DiceRoll":"1d4"},"Stats":{"Strength":{"Base":3},"Speed":{"Base":6},"Vitality":{"Base":3},"Smarts":{"Base":2},"Mysticism":{"Base":1},"Perception":{"Base":5}}}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/races
200 Success
{"success":true,"data":{"RaceId":42,"Name":"Goblin",...}}
500 Validation error
{"success":false,"error":"race has no description"}
PATCH /admin/api/v1/races/{raceId} Update a race

{raceId} must be a numeric race ID. Fields in the request body overwrite the existing race. The RaceId is preserved.

curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{"Name":"Human","Description":"Updated description.","Size":"medium","TNLScale":1.0}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/races/1
200 Success
{"success":true,"data":{"RaceId":1,"Name":"Human",...}}
404 Not found
{"success":false,"error":"race not found: 99"}
DELETE /admin/api/v1/races/{raceId} Delete a race

{raceId} must be a numeric race ID. Removes the race data file from disk.

curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/races/42
200 Success
{"success":true,"data":null}
404 Not found
{"success":false,"error":"race not found: 42"}
