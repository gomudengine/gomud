Quests API Reference

REST endpoints for listing, creating/updating, and deleting quest definitions.

GET /admin/api/v1/quests Return all quest definitions

Returns an array of every loaded quest, including secret quests.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/quests
200 Success
{"success":true,"data":[{"QuestId":1001,"Name":"The Lost Artifact","Secret":false,"Steps":[{"Id":"start","Description":"Speak to the archaeologist","Hint":""}],"Rewards":{"Gold":100,"Experience":500}}]}
PATCH /admin/api/v1/quests Create or update a quest

Upserts a quest. The QuestId field in the body is required and must be a positive integer. If the quest already exists it is replaced; if it does not exist it is created. The quest is saved to disk immediately.

Quest step tokens follow the format {QuestId}-{stepId}, e.g. 1001-start.

curl -s -u admin:password \ -X PATCH \ -H "Content-Type: application/json" \ -d '{ "QuestId": 1001, "Name": "The Lost Artifact", "Description": "Recover the ancient relic.", "Secret": false, "Steps": [ {"Id": "start", "Description": "Speak to the archaeologist"}, {"Id": "search", "Description": "Search the ancient ruins"}, {"Id": "end", "Description": "Return the artifact"} ], "Rewards": { "Gold": 100, "Experience": 500, "PlayerMessage": "You have recovered the artifact!" } }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/quests
200 Success
{"success":true,"data":{"QuestId":1001,"Name":"The Lost Artifact",...}}
400 Bad Request
{"success":false,"error":"questId must be set in the request body"}
DELETE /admin/api/v1/quests/{questId} Delete a quest

Permanently removes the quest YAML from disk and from the in-memory cache. {questId} must be a numeric quest ID.

curl -s -u admin:password \ -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/quests/1001
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"quest 1001 not found"}
