Users API Reference

REST endpoints for searching and managing user accounts.

GET /admin/api/v1/users/search Search users by username

Searches for user accounts whose username matches or starts with the provided name query parameter. The search is case-insensitive.

If an exact match is found, the response contains exactly one entry. Otherwise all prefix matches are returned. An empty array is returned when no matches exist.

ParameterInDescription
name required query Username (or prefix) to search for.
curl -s -u admin:password \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/users/search?name=alice"
200 Success — exact match

Exactly one result when the username matches precisely.

{ "success": true, "data": [ { "user_id": 42, "username": "alice", "role": "user", "email": "alice@example.com" } ] }
200 Success — prefix matches

Multiple results when the query matches the start of several usernames.

{ "success": true, "data": [ { "user_id": 42, "username": "alice", "role": "user", "email": "" }, { "user_id": 57, "username": "alicebot", "role": "guest", "email": "" } ] }
200 Success — no results
{"success":true,"data":[]}
400 Bad Request

Returned when the name query parameter is missing.

{"success":false,"error":"name query parameter is required"}
GET /admin/api/v1/users/{userid} Get a user by ID

Returns the full UserRecord for the given numeric user ID. Online users are served from the in-memory cache; offline users are loaded from disk.

ParameterInDescription
userid required path Numeric user ID.
curl -s -u admin:password \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/users/42"
200 Success
{ "success": true, "data": { "userid": 42, "role": "user", "username": "alice", "password": "$2a$10$...", "joined": "2024-01-15T10:30:00Z", "emailaddress": "alice@example.com", "muted": false, "screenreader": false } }
400 Bad Request

Returned when the userid path segment is not a valid integer.

{"success":false,"error":"invalid user id: abc"}
404 Not Found
{"success":false,"error":"user not found"}
PATCH /admin/api/v1/users/{userid} Update a user record

Merges the supplied JSON fields into the existing UserRecord and persists the result to disk. Only fields present in the request body are applied. The UserId is always preserved from the URL and cannot be changed.

To change a password use SetPassword via the in-game admin commands; the password field written here is stored as-is and must already be a valid bcrypt hash if you supply it.

ParameterInDescription
userid required path Numeric user ID.
curl -s -u admin:password \ -X PATCH \ -H "Content-Type: application/json" \ -d '{"role":"admin","emailaddress":"newemail@example.com"}' \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/users/42"
200 Success

Returns the full updated UserRecord.

{ "success": true, "data": { "userid": 42, "role": "admin", "username": "alice", "emailaddress": "newemail@example.com", ... } }
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"user not found"}
POST /admin/api/v1/users Create a new user

Creates a new user account. A fresh UserRecord is initialised with NewUserRecord, the supplied fields are applied, and the record is saved. The UserId is assigned automatically.

The request will be rejected with 409 Conflict if the username is already taken.

FieldTypeDescription
Username required string Desired username. Must pass server name-validation rules.
Password string Plain-text password. Hashed with bcrypt before storage. Optional — account will have no password if omitted.
Role string Account role (user, admin, etc.). Defaults to user when omitted.
EmailAddress string Optional email address.
curl -s -u admin:password \ -X POST \ -H "Content-Type: application/json" \ -d '{"Username":"bob","Password":"s3cr3t","Role":"user","EmailAddress":"bob@example.com"}' \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/users"
201 Created

Returns the newly created UserRecord.

{ "success": true, "data": { "userid": 101, "role": "user", "username": "bob", "emailaddress": "bob@example.com", "joined": "2026-04-23T12:00:00Z", ... } }
400 Bad Request

Returned when the body is malformed, Username is missing, or the name fails validation.

{"success":false,"error":"Username is required"}
409 Conflict

Returned when the username is already in use.

{"success":false,"error":"username already exists"}
