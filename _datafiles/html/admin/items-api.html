Items API Reference

REST endpoints for browsing, creating, editing, and deleting item definitions.

GET /admin/api/v1/items/types Return all item types and subtypes

Returns the full list of item types (with ID ranges) and subtypes (with descriptions).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/types
200 Success
{"success":true,"data":{"subtypes":[...],"types":[{"Type":"weapon","Description":"...","Count":0,"MinItemId":10000,"MaxItemId":19999},...]}}
GET /admin/api/v1/items/attack-messages Return all weapon attack message groups

Returns all loaded weapon attack message groups keyed by subtype (generic, slashing, stabbing, etc.).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/attack-messages
GET /admin/api/v1/items Return all item specs

Returns an array of every loaded item specification.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items
POST /admin/api/v1/items Create a new item

Creates a new item. The ItemId is assigned automatically based on Type. Returns the created spec.

curl -s -u admin:password \ -X POST \ -H "Content-Type: application/json" \ -d '{"Name":"Rusty Dagger","Type":"weapon","Subtype":"stabbing","Damage":{"DiceRoll":"1d4"},"Hands":1}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items
201 Created
{"success":true,"data":{"ItemId":10042,"Name":"Rusty Dagger",...}}
400 Bad Request
{"success":false,"error":"Could not find a new item id to assign."}
GET /admin/api/v1/items/{itemId} Get a single item spec by ID or name

{itemId} can be a numeric item ID or a partial/full item name (uses items.FindItem() matching).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/10042 curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/rusty%20dagger
PATCH /admin/api/v1/items/{itemId} Update item properties

Merges the supplied fields into the existing spec. The ItemId in the body is ignored — the path parameter is authoritative.

curl -s -u admin:password \ -X PATCH \ -H "Content-Type: application/json" \ -d '{"Description":"A corroded blade."}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/10042
DELETE /admin/api/v1/items/{itemId} Delete an item and its script

Permanently removes the item YAML and associated JS script from disk and from the in-memory cache.

curl -s -u admin:password \ -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/10042
GET /admin/api/v1/items/{itemId}/script Get the item's JavaScript

Returns the raw script content. script is an empty string if no script file exists.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/10042/script
200 Success
{"success":true,"data":{"script":"function onFound(user,item,room){...}"}}
PUT /admin/api/v1/items/{itemId}/script Replace (or delete) the item's JavaScript

Writes the provided script string to the item's .js file. Send an empty string to delete the script file.

curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"script":"function onFound(user,item,room){}"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/items/10042/script
