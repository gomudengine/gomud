Panel Layouts API Reference

REST endpoints for reading, validating, previewing, and saving panel layout YAML files.

YAML Format Reference

A panel layout file is a YAML document that describes how a set of bordered panels are arranged on screen. The file is loaded by Go code which then populates each panel with rows of data before rendering.

Top-level fields

Field Values / Notes
border "full" — every content row has side borders  │ label  value │
"open" — only the first and last content rows have side borders
Default: "full"
charset "single" — ┌─┐ │ └─┘  (default)
"double" — ╔═╗ ║ ╚═╝
"rounded" — ╭─╮ │ ╰─╯
8-rune literal in the order: TopLeft HorizTop TopRight VertLeft VertRight BottomLeft HorizBottom BottomRight
Example: "╔═╗║║╚═╝"
gap Number of spaces between panels placed side by side and between top-level slots.
margin Spaces prepended to every output line. Optional, default 0.
default_color Optional ANSI fg color that wraps the entire rendered output in a single <ansi> tag.
Accepts a numeric 256-color code (e.g. "214") or a named alias (e.g. "gold").
Omit or leave blank to disable.

Layout model

Panels are arranged in a three-level hierarchy:

  • slots — top-level horizontal columns, rendered side by side
  • rows — within a slot, panel groups stacked vertically top-to-bottom
  • panels — within a row, individual panels placed side by side left-to-right

Per-panel fields

Field Values / Notes
id Unique string identifier. Used in Go code: layout.Panel("id").Add(...). Required.
title Rendered verbatim into the top border. May contain ANSI tags — visual width is measured with tags stripped. Omit for a plain border line with no title gap.
min_width Minimum inner content width in characters. Expands automatically if content is wider. Total panel width = min_width + 4 (two border chars + two padding spaces).
columns Number of label+value pairs rendered per line. Optional, default 1.
column_gap Spaces between columns when columns > 1. Optional, default 2.
charset Per-panel charset override. Accepts the same values as the top-level charset. Falls back to the top-level charset when omitted.

Minimal example

border: open
charset: single
gap: 1
margin: 1

slots:
  - rows:
      - panels:
          - id: info
            title: ' Info '
            min_width: 30
  - rows:
      - panels:
          - id: stats
            title: ' Stats '
            min_width: 40
            columns: 2
            column_gap: 2
GET /admin/api/v1/panels Return all panel layout files panels.read

Returns an array of every panel layout file found under the panel-layouts/ data directory. Each entry includes the relative name (e.g. character/status) and the raw yaml content.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/panels
200 Success
{"success":true,"data":[{"name":"character/status","yaml":"border: open\ncharset: single\n..."}]}
POST /admin/api/v1/panels/validate Validate a panel layout YAML panels.write

Parses the supplied YAML and checks for structural problems: unknown border or charset values, missing or duplicate panel IDs, empty slots or rows, and negative numeric fields. Returns valid: true and an empty issues array when the layout is clean.

curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"yaml":"border: open\ncharset: single\ngap: 1\nslots:\n - rows:\n - panels:\n - id: info\n title: Info\n min_width: 25\n"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/panels/validate
200 Valid layout
{"success":true,"data":{"valid":true,"issues":[]}}
200 Invalid layout (issues found)
{"success":true,"data":{"valid":false,"issues":[{"message":"slot 1, row 1, panel 1: missing id"},{"message":"unknown border style \"fancy\" (valid: \"full\", \"open\")"}]}}
400 YAML parse error
{"success":false,"error":"invalid YAML: yaml: line 3: ..."}
POST /admin/api/v1/panels/preview Render a plain-text preview of a panel layout panels.write

Parses the supplied YAML, fills every panel with dummy rows, and returns the rendered plain-text output. ANSI tags in titles are stripped so the result is safe to display in a browser. Sole panels in a row receive 4 dummy rows; panels with horizontal siblings receive 2.

curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"yaml":"border: open\ncharset: single\ngap: 1\nslots:\n - rows:\n - panels:\n - id: info\n title: Info\n min_width: 25\n"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/panels/preview
200 Success
{"success":true,"data":{"preview":"┌─ Info ───────────────────┐\n│ label-1 value-1 │\n..."}}
400 YAML parse error
{"success":false,"error":"preview panel layout: yaml: ..."}
PUT /admin/api/v1/panels/{name} Overwrite an existing panel layout file panels.write

Overwrites the panel layout file identified by {name} (e.g. character/status). The file must already exist; this endpoint does not create new layout files. The YAML is validated for parse errors before writing.

Encode slashes in the name as %2F or pass the full path directly — the route uses a wildcard catch-all so nested paths are preserved.

curl -s -u admin:password -X PUT \ -H "Content-Type: application/json" \ -d '{"yaml":"border: open\ncharset: single\n..."}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/panels/character/status
200 Success
{"success":true}
400 File not found or invalid YAML
{"success":false,"error":"panel layout \"character/status\": stat ...: no such file or directory"}
