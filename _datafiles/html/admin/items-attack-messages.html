Available Tokens
Token Resolves To Context
{source} Attacker's display name Always available. Use in messages shown to the defender or the room so they know who is attacking.
always set
{sourcetype} username or mobname Always available. Used as an ANSI color tag foreground value to color the attacker's name correctly, e.g. <ansi fg="{sourcetype}">{source}</ansi>.
always set
{target} Defender's display name Always available. Use in messages shown to the attacker or the room so they know who is being hit.
always set
{targettype} username or mobname Always available. Used as an ANSI color tag foreground value to color the defender's name correctly, e.g. <ansi fg="{targettype}">{target}</ansi>.
always set
{itemname} Weapon display name, or unarmed attack name Always available. Falls back to the race's unarmed attack name when no weapon is equipped. Useful in all message targets to describe what was used.
always set
{damage} Numeric damage dealt (e.g. 14) Available for all non-prepare, non-wait, non-miss intensities. Not meaningful in prepare or wait messages since no damage has been dealt yet. Best used in To Attacker messages for feedback.
always set
{exitname} Exit direction from attacker's room toward defender (e.g. north) Only meaningful in Separate Rooms messages. Resolves to unknown in same-room combat. Use in To Attacker or To Attacker Room to indicate which direction the attack was fired.
separate rooms only
{entrancename} Exit direction from defender's room back toward attacker (e.g. south) Only meaningful in Separate Rooms messages. Resolves to unknown in same-room combat. Use in To Defender or To Defender Room to show where the attack came from.
separate rooms only
