Mobs API Reference

REST endpoints for browsing, creating, editing, and deleting mob definitions. ← Back to Mobs

GET /admin/api/v1/mobs Return all mob specs

Returns an array of every loaded mob specification. Each entry includes a HasScript boolean indicating whether a JavaScript file exists for the mob.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs
200 Success
{"success":true,"data":[{"MobId":1,"Zone":"Frostfang","Hostile":false,"HasScript":true,"Character":{"Name":"rat","Level":1,...}},...]}
POST /admin/api/v1/mobs Create a new mob

Creates a new mob. The MobId is assigned automatically (next available). Returns the created spec.

curl -s -u admin:password \ -X POST \ -H "Content-Type: application/json" \ -d '{"Zone":"Frostfang","Hostile":true,"ActivityLevel":20,"Character":{"Name":"goblin","Level":5,"RaceId":10}}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs
201 Created
{"success":true,"data":{"MobId":42,"Zone":"Frostfang","Hostile":true,"Character":{"Name":"goblin",...}}}
400 Bad Request
{"success":false,"error":"Could not find a new mob id to assign."}
GET /admin/api/v1/mobs/{mobId} Get a single mob spec by ID or name

{mobId} can be a numeric mob ID or a partial/full mob name (uses fuzzy matching).

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/5 curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/armorer
200 Success
{"success":true,"data":{"MobId":5,"Zone":"Frostfang","Character":{"Name":"armorer","Level":29,...}}}
404 Not Found
{"success":false,"error":"mob not found: 99999"}
PATCH /admin/api/v1/mobs/{mobId} Update mob properties

Merges the supplied fields into the existing spec. The MobId in the body is ignored — the path parameter is authoritative. The mob YAML file is overwritten and the in-memory cache is updated.

curl -s -u admin:password \ -X PATCH \ -H "Content-Type: application/json" \ -d '{"Hostile":false,"ActivityLevel":30}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/5
200 Success
{"success":true,"data":{"MobId":5,"Zone":"Frostfang","Hostile":false,"ActivityLevel":30,...}}
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"mob not found: 99999"}
DELETE /admin/api/v1/mobs/{mobId} Delete a mob and its script

Permanently removes the mob YAML and associated JS script from disk and from the in-memory cache.

curl -s -u admin:password \ -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/42
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"mob not found: 99999"}
GET /admin/api/v1/mobs/{mobId}/script Get the mob's JavaScript

Returns the raw script content. script is an empty string if no script file exists. The script filename is derived from the mob's ID, name, and optional ScriptTag.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/26/script
200 Success
{"success":true,"data":{"script":"function onAsk(mob,room,eventDetails){...}"}}
PUT /admin/api/v1/mobs/{mobId}/script Replace (or delete) the mob's JavaScript

Writes the provided script string to the mob's .js file. Send an empty string to delete the script file. The script directory is created automatically if it does not exist.

curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"script":"function onAsk(mob,room,eventDetails){}"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/26/script
200 Success
{"success":true}
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"mob not found: 99999"}
PUT /admin/api/v1/mobs/{mobId}/stock Add an entry to the mob's shop

Appends a shop entry to the mob's shop inventory and saves to disk. The entry can be an item, mob-for-hire, buff, or pet. Set exactly one of ItemId, MobId, BuffId, or PetType.

QuantityMax values: 0 = always in stock (unlimited), -1 = temporary (never restocks, removed when depleted), >0 = restocks up to that amount.

Optional fields: Price (0 = auto from item value), TradeItemId (item required in trade), RestockRate (e.g. "1 hour", "1 day", "1 week").

curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"ItemId":20002,"QuantityMax":3,"RestockRate":"1 hour"}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/5/stock
curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"BuffId":14,"Price":50,"QuantityMax":0}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/4/stock
curl -s -u admin:password \ -X PUT \ -H "Content-Type: application/json" \ -d '{"PetType":"cat","Price":1200,"QuantityMax":0}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mobs/44/stock
200 Success
{"success":true}
400 Bad Request
{"success":false,"error":"malformed request body: ..."}
404 Not Found
{"success":false,"error":"mob not found: 99999"}
