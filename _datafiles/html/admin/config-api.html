Config API Reference

REST endpoints for reading and updating server configuration.

GET /admin/api/v1/config Return all configuration values as a flat key/value map

Returns the full server configuration as a flat dot-path key/value map. Secrets are redacted.

curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/config
200 Success

Returns the full configuration map. Secrets appear as *** REDACTED ***.

{ "success": true, "data": { "Server.MudName": "GoMud", "Server.Seed": "*** REDACTED ***", "Network.HttpPort": "80" } }
401 Unauthorized

Invalid or missing credentials.

{ "success": false, "error": "unauthorized" }
PATCH /admin/api/v1/config Update one or more configuration values

Accepts a JSON object of dot-path config keys to new string values. Locked keys are rejected and returned in the rejected list. Unknown keys return 400.

curl -s -u admin:password \ -X PATCH \
-H "X-Test-Mode: true" \ -H "Content-Type: application/json" \ -d '{ "Server.MudName": "My MUD" }' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/config
200 Success

applied lists keys that were updated. rejected lists locked keys that were silently skipped.

{ "success": true, "data": { "applied": ["Server.MudName"], "rejected": ["Server.Seed"] } }
400 Bad Request

Returned when an unknown config key is supplied or the request body is malformed.

{ "success": false, "error": "unknown config key: Server.Typo" }
401 Unauthorized

Invalid or missing credentials.

{ "success": false, "error": "unauthorized" }
