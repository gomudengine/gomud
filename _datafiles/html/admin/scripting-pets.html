Pet Scripts

Animate companion pets with periodic actions, command interception, and departure/return events.

Script Path

Pet scripts live in a scripts/ subdirectory next to the pet YAML definition:

Pet definition _datafiles/world/default/pets/dog.yaml
Pet script _datafiles/world/default/pets/scripts/dog.js

Script State

Global variables in a pet script are shared across all owners of that pet type. For per-owner data, use actor.SetTempData() or actor.SetMiscCharacterData() on the owner actor.

Event Hooks

PetAct(pet PetObject, actor ActorObject, room RoomObject)

Called each round with a probability set by the pet type's RoundActChance (0–100). Not called while the owner is in combat. The engine handles the probability check before calling this function — you don't need a top-level random guard unless you want behavior that fires less often than the configured chance.

ArgumentTypeDescription
petPetObjectThe pet.
actorActorObjectThe owner.
roomRoomObjectThe room both are in.
function PetAct(pet, actor, room) {
    // 5% chance per eligible round to do something visible
    if (Math.random() < 0.05) {
        room.SendText(pet.NameSimple() + ' sniffs the air curiously.');
    }
}
onCommand(cmd string, rest string, pet PetObject, actor ActorObject, room RoomObject)

Called whenever the pet's owner types any command. Fires after buff onCommand handlers and before item onCommand handlers. Return true to consume the command.

Return value: true = consumed. false or nothing = pass through.
ArgumentTypeDescription
cmdstringThe command word typed by the owner.
reststringEverything after the command word.
petPetObjectThe pet.
actorActorObjectThe owner.
roomRoomObjectThe current room.
function onCommand(cmd, rest, pet, actor, room) {
    if (cmd === 'attack') {
        if (Math.random() < 0.25) {
            room.SendText(pet.NameSimple() + ' growls menacingly.');
        }
    }
    return false;
}
onCommand_{command}(rest string, pet PetObject, actor ActorObject, room RoomObject)

Targeted variant — fires only when the owner types the specific command named after the underscore. When defined, generic onCommand is not called for that command.

Return value: true = consumed. false or nothing = pass through.
ArgumentTypeDescription
reststringEverything after the command word.
petPetObjectThe pet.
actorActorObjectThe owner.
roomRoomObjectThe current room.
// Called only when owner types 'pet'
function onCommand_pet(rest, pet, actor, room) {
    room.SendText(pet.NameSimple() + ' wags its tail happily.');
    return false;
}
PetLeave(pet PetObject, actor ActorObject, room RoomObject)

Called immediately when PetObject.GoMissing() is invoked. The pet is already marked as missing when this fires (pet.IsMissing() returns true).

ArgumentTypeDescription
petPetObjectThe departing pet.
actorActorObjectThe owner.
roomRoomObjectThe room both are in.
function PetLeave(pet, actor, room) {
    room.SendText(pet.NameSimple() + ' darts into the shadows and disappears!');
}
PetReturn(pet PetObject, actor ActorObject, room RoomObject)

Called the round the pet's MissingCountdown reaches zero. The countdown has already cleared when this fires (pet.IsMissing() returns false).

ArgumentTypeDescription
petPetObjectThe returning pet.
actorActorObjectThe owner.
roomRoomObjectThe room both are in.
function PetReturn(pet, actor, room) {
    room.SendText(pet.NameSimple() + ' trots back to your side.');
}
