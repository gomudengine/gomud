Audio API Reference

REST endpoints for reading and managing audio configuration entries.

GET /admin/api/v1/audio Return sounds, music, and available sound files

Returns three sections:

  • sounds — map of identifier → audio config loaded from audio.yaml. Each entry has an optional description, a filepath (relative to WebCDNLocation, or host root if blank), and an optional volume (0–100).
  • music — sorted list of filenames found in PublicHtml/static/audio/music/. Files beginning with _ are excluded. Read-only.
  • soundFiles — sorted list of all files found recursively under PublicHtml/static/audio/sound/. Files beginning with _ are excluded. Read-only.
curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/audio
200 Success
{"success":true,"data":{"sounds":{"intro":{"description":"Plays at login screen","filepath":"static/audio/music/intro.mp3","volume":20},"levelup":{"description":"When leveling up","filepath":"static/audio/sound/other/levelup.mp3"}},"music":["static/audio/music/intro.mp3"],"soundFiles":["static/audio/sound/combat/hit-other.mp3","static/audio/sound/other/levelup.mp3"]}}
POST /admin/api/v1/audio Create a new sound entry audio.write

Creates a new sound identifier. Returns 409 Conflict if the identifier already exists.

Body fields:

  • identifier (string, required) — unique key used in code to trigger the sound.
  • description (string, optional) — human-readable description.
  • filepath (string, optional) — path relative to WebCDNLocation.
  • volume (integer 0–100, optional) — base volume; 0 means use 100%.
  • tags (array of strings, optional) — free-form labels describing where this sound is used.
curl -s -u admin:password -X POST \ -H "Content-Type: application/json" \ -d '{"identifier":"my-sound","description":"Custom sound","filepath":"static/audio/sound/other/change.mp3","volume":80}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/audio
200 Success — returns full updated audio data
{"success":true,"data":{"sounds":{...},"music":[...],"soundFiles":[...]}}
400 Bad Request — missing identifier
{"success":false,"error":"identifier is required"}
409 Conflict — identifier already exists
{"success":false,"error":"identifier already exists: my-sound"}
PATCH /admin/api/v1/audio Replace the entire sounds configuration audio.write

Replaces the entire sounds configuration with the provided map. The body must be a JSON object mapping identifiers to audio config objects. The updated configuration is written to audio.yaml and reloaded in memory.

The music and soundFiles lists are derived from disk and cannot be modified through this endpoint.

curl -s -u admin:password -X PATCH \ -H "Content-Type: application/json" \ -d '{"intro":{"description":"Plays at login screen","filepath":"static/audio/music/intro.mp3","volume":20}}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/audio
200 Success
{"success":true,"data":{"sounds":{...},"music":[...],"soundFiles":[...]}}
400 Bad Request
{"success":false,"error":"malformed request body: unexpected EOF"}
DELETE /admin/api/v1/audio/{identifier} Delete a single sound entry audio.write

Removes the named sound identifier from audio.yaml and reloads the in-memory configuration. Returns 404 if the identifier does not exist.

curl -s -u admin:password -X DELETE \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/audio/my-sound
200 Success
{"success":true}
404 Not Found
{"success":false,"error":"identifier not found: my-sound"}
