Combat Message Tokens

Each message field supports the following tokens, which are replaced at runtime:

TokenReplaced with
{petname}The pet’s display name (may include ANSI colour tags).
{damage}The integer damage dealt (plain text, e.g. 12).
{targetname}The target’s formatted name (includes ANSI colour tags matching the target type).

Message slots

To Owner — sent to the player who owns the pet when the attack lands.

To Target — sent to the player being attacked (only when the target is a player).

To Room — sent to everyone else in the room observing the attack.

Miss — sent to the owner when the pet’s attack misses. {damage} is 0 in miss messages.

Leave any field empty to use the built-in default message for that slot. Messages are only shown when the ability has a damage dice roll defined.

Examples

{petname} leaps at {targetname} and bites for {damage} damage!
{petname} snaps at {targetname} but finds only air.
You feel a sudden pain as {petname} sinks its teeth into you for {damage} damage!
