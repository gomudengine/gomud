Mudmail API Reference

REST endpoints for managing player inbox messages.

GET /admin/api/v1/mudmail?user_id={id} List message summaries for one user

Returns summary entries (no message body) for every message in the specified user's inbox. The user_id query parameter is required.

Query parameters:

  • user_id (required) — the recipient's user ID
curl -s -u admin:password \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mudmail?user_id=42"
200 Success

Returns a messages array of summary objects. Each entry contains:

  • user_id — recipient user ID
  • username — recipient username
  • from_name — sender display name
  • read — whether the player has read this message
  • has_gold — true if gold is attached (omitted when false)
  • has_item — true if an item is attached (omitted when false)
  • date_sent — RFC3339 timestamp
  • date_sent_us — microseconds since Unix epoch (key for body and delete endpoints)
{ "success": true, "data": { "messages": [ { "user_id": 42, "username": "alice", "from_name": "System", "read": false, "has_gold": true, "date_sent": "2025-01-15T10:30:00.123456Z", "date_sent_us": 1736937000123456 } ] } }
400 Bad Request

Returned when user_id is missing or invalid.

{ "success": false, "data": { "error": "user_id query param is required" } }
401 Unauthorized
{ "success": false, "error": "unauthorized" }
GET /admin/api/v1/mudmail-body/{user_id}/{timestamp} Fetch full message data for one message

Returns the complete message including the body text, gold, and read status. Use the date_sent_us value from the summary endpoint as the {timestamp} path segment.

Path parameters:

  • user_id — the recipient's user ID
  • timestamp — microseconds since Unix epoch of the message (from date_sent_us in the summary response)
curl -s -u admin:password \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mudmail-body/42/1736937000123456
200 Success

Returns the full message object:

  • user_id, username, from_name, read, date_sent, date_sent_us — same as summary
  • body — full message text
  • gold — attached gold amount (omitted if 0)
{ "success": true, "data": { "user_id": 42, "username": "alice", "from_name": "System", "body": "Welcome to the server!", "gold": 100, "read": false, "date_sent": "2025-01-15T10:30:00.123456Z", "date_sent_us": 1736937000123456 } }
400 Bad Request

Returned when path parameters are missing or invalid.

{ "success": false, "data": { "error": "invalid user_id" } }
404 Not Found
{ "success": false, "data": { "error": "message not found" } }
401 Unauthorized
{ "success": false, "error": "unauthorized" }
POST /admin/api/v1/mudmail Send a message to one player or everyone

Request body fields:

  • from_name (required) — sender display name
  • body (required) — message text
  • gold (optional) — gold to attach (default 0)
  • user_id (optional) — recipient user ID; omit or set to 0 to broadcast to all players

Send to one player:

curl -s -u admin:password \ -X POST \ -H "Content-Type: application/json" \ -d '{"from_name":"System","body":"Hello!","user_id":42}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mudmail

Broadcast to everyone:

curl -s -u admin:password \ -X POST \ -H "Content-Type: application/json" \ -d '{"from_name":"System","body":"Server maintenance in 10 minutes."}' \ http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mudmail
200 Sent to one player
{ "success": true, "data": { "sent_to": 42 } }
200 Broadcast to all
{ "success": true, "data": { "sent_to_all": true, "recipient_count": 127 } }
400 Bad Request

Returned when from_name or body is missing, or the JSON is malformed.

{ "success": false, "data": { "error": "body is required" } }
401 Unauthorized
{ "success": false, "error": "unauthorized" }
DELETE /admin/api/v1/mudmail Delete one message by user ID and send timestamp

Query parameters:

  • user_id (required) — the recipient's user ID
  • date_sent_us (required) — the microseconds-since-epoch timestamp of the message (from date_sent_us in the summary response)
curl -s -u admin:password \ -X DELETE \ "http://{{.CONFIG.FilePaths.WebDomain}}/admin/api/v1/mudmail?user_id=42&date_sent_us=1736937000123456"
200 Deleted
{ "success": true, "data": { "deleted": true } }
400 Bad Request

Returned when required query params are missing or invalid.

{ "success": false, "data": { "error": "user_id and date_sent_us query params are required" } }
404 Not Found

No message matched the provided user ID and timestamp.

{ "success": false, "data": { "error": "message not found" } }
401 Unauthorized
{ "success": false, "error": "unauthorized" }
