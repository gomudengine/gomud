Discord Activity

Version 1.0 — Embed GoMud as a playable Discord Activity inside voice channels.

What This Module Does

The Discord Activity module serves the full GoMud web client as a Discord Embedded App. Players can connect to the MUD directly inside a Discord voice channel by clicking the rocket (Activities) icon, without leaving Discord.

The Activity is functionally identical to the standard web client: same WebSocket endpoint, same xterm.js terminal, GMCP, audio, triggers, and virtual window panels. Players log in with their normal MUD username and password — no Discord account linking is required.

Virtual window panels (character stats, map, gear, etc.) are hidden by default to make the best use of the compact Activity frame. They become visible automatically when the user expands the Activity to full-screen using Discord's expand button.

Part 1 — Create the Discord Application

These steps are done once in the Discord Developer Portal. You need a Discord account with Developer Mode enabled.

Enable Developer Mode on Your Account

Discord's Activity shelf only appears for accounts that have Developer Mode enabled.

  1. 1
    Open Discord and go to User Settings (the gear icon next to your username in the bottom-left).
  2. 2
    Click Advanced in the left sidebar and toggle Developer Mode on.
    This setting is per-account. Each person who wants to test the Activity in Discord must enable it on their own account.
Create the Application
  1. 1
    Go to discord.com/developers/applications and click New Application.
  2. 2
    Give it a name (e.g. the name of your MUD). Accept the terms and click Create.
  3. 3
    On the General Information page, copy the Application ID. You will need this later.
    The Application ID is a long number, e.g. 1234567890123456789. It is public and safe to store in config.
Enable Activities and Configure URL Mapping
  1. 1
    In the left sidebar of the Developer Portal, click Activities.
  2. 2
    Click Enable Activities to turn on the Activities feature for this application.
    This makes the application visible in Discord's Activity shelf and unlocks the URL Mapping settings.
  3. 3
    Under URL Mappings, click Add a mapping and enter the following:
    PrefixTarget
    / your-server-domain.com:443
    Replace your-server-domain.com:443 with the hostname and HTTPS port of your GoMud server. The exact value to use is shown on the Config page under "Activity Host".
    Do not include a protocol (https://) in the Target field. Discord adds the protocol automatically. Targets must point to a directory root, not a specific file.
  4. 4
    Save the URL mapping. Discord will now proxy all Activity requests through to your GoMud server.
Set Up the Entry Point (Required to Appear in the Shelf)

Discord requires an "Entry Point Command" for the Activity to appear in the Activity shelf inside voice channels.

  1. 1
    Still under Activities in the Developer Portal, scroll down to Default Entry Point Command.
  2. 2
    Set the handler to Activity (not a slash command). This tells Discord to open your Activity URL directly when a user clicks the Activity in the shelf.
    If this section is not visible, ensure you have clicked Enable Activities first.
Add a Redirect URI (Required for the SDK Handshake)
  1. 1
    In the left sidebar, click OAuth2.
  2. 2
    Under Redirects, click Add Redirect and enter https://your-server-domain.com/discord-activity.
    Use the same domain as your GoMud server's HTTPS address. This is required by the Discord SDK handshake even though this module does not perform OAuth2 user login.
  3. 3
    Click Save Changes.

Part 2 — Configure GoMud

  1. 1
    Go to the Discord Activity Config page in this admin panel.
  2. 2
    Paste the Application ID you copied from the Developer Portal into the Application ID field and click Save.
  3. 3
    Confirm that GoMud is running with HTTPS configured (Network.HttpsPort set in server config). Discord will refuse to load an Activity served over plain HTTP.
    If the Config page shows a warning about HTTPS not being configured, set up TLS before proceeding. See the HTTPS section in the admin settings.

Part 3 — Add the Application to a Discord Server

Before the Activity appears in a server's Activity shelf, the application must be installed into that server.

  1. 1
    In the Developer Portal, go to Installation in the left sidebar.
  2. 2
    Under Install Link, copy the generated OAuth2 install URL (or the Discord-provided install link).
  3. 3
    Open the install URL in a browser while logged into Discord. Select the server you want to add the Activity to and click Authorize.
    You need Manage Server (or Administrator) permission in the target server to install an application.

Part 4 — Testing the Activity

Launching in Discord Desktop
  1. 1
    Join any voice channel in a server where the application has been installed.
  2. 2
    In the voice channel controls (the panel that appears at the bottom or side), click the rocket icon to open the Activity shelf.
    The rocket icon only appears when Developer Mode is enabled on your account and at least one Activity-capable application is installed in the server.
  3. 3
    Find your MUD application in the shelf and click it. The Activity will load in an embedded panel showing the MUD terminal.
  4. 4
    Click the expand / full-screen button (the arrows icon in the Activity frame's top-right corner) to switch to full-screen mode. The virtual window panels (map, character stats, gear, etc.) will appear.
  5. 5
    Log in with your normal MUD username and password at the terminal prompt.
Launching on Discord Mobile
  1. 1
    From your User Profile, tap Appearance and toggle Developer Mode on.
  2. 2
    Join a voice channel and tap the Activity button to open the shelf, then select your application.
Viewing Debug Logs

If the Activity does not load or shows an error, use Discord's built-in developer tools to diagnose the problem.

  1. 1
    On desktop, open the Activity and press Ctrl+Shift+I (Windows/Linux) or Cmd+Option+I (Mac) to open the browser developer tools inside the Activity frame.
    Alternatively, in Discord's top menu go to View → Developer → Toggle Developer Tools.
  2. 2
    Check the Console tab for errors. Activity-specific log messages are prefixed with [RpcApplicationLogger] followed by your Application ID.
  3. 3
    Check the Network tab to confirm that requests to /discord-activity and /ws are succeeding. Failed requests here usually mean the URL Mapping target is incorrect or the server is not reachable over HTTPS.

Troubleshooting

Activity does not appear in the rocket shelf.
Confirm Developer Mode is enabled on your account, the application is installed in the server, and the Entry Point Command handler is set to Activity in the Developer Portal.

Activity loads but shows a blank page or "not configured" error.
The Application ID has not been saved in GoMud. Go to the Config page and save the correct Application ID.

Activity loads but the SDK handshake fails (console shows a ready() error).
The Application ID in GoMud config does not match the one registered in the Discord Developer Portal, or the Redirect URI has not been added under OAuth2 in the portal.

Activity shows an error about HTTP / mixed content.
GoMud must be served over HTTPS. Configure Network.HttpsPort and ensure a valid TLS certificate is in place.

WebSocket connection fails inside the Activity.
Verify the URL Mapping in the Developer Portal has the correct Target (no https:// prefix, correct hostname and port). The target must resolve to your GoMud server's HTTPS address.

Virtual window panels do not appear.
Panels are intentionally hidden in compact mode. Click the expand/full-screen button in the Activity frame's top-right corner to enter full-screen mode.

Technical Notes

The Activity page is served at /discord-activity on the GoMud HTTPS server. It uses the Discord Embedded App SDK loaded from CDN — no Node.js, npm, or build step is required.

Discord proxies all HTTP and WebSocket traffic through its own proxy layer (*.discordsays.com). The SDK's patchUrlMappings() call rewrites outbound connections transparently, so the existing /ws WebSocket endpoint works without any server-side changes.

The Activity HTML is based on webclient-pure.html. If core webclient functionality changes in that file, the same changes should be applied to discord-activity.html in this module's files/datafiles/html/public/ directory.

Room Tags: This module does not use any room tags.
